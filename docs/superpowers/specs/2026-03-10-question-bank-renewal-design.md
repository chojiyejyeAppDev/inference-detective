# 문제 뱅크 전체 리뉴얼 설계

## 배경

고2 학생 관점에서 전 문제를 풀어본 결과, 오답 카드가 극단적 한정어("반드시", "완전히", "모든")나 지문과 정반대 내용으로 구성되어 지문을 읽지 않고도 제거할 수 있었다. 정답 카드의 id도 지문 순서와 일치해 알파벳순 정렬만으로 풀 수 있었다. 추론 능력 훈련이라는 앱의 목적에 비해 변별력이 부족하므로 전체 리뉴얼이 필요하다.

## 설계 원칙

### 1. 오답 카드 — 레벨별 차등

| 레벨 | 오답 유형 | 오답 수 | 설명 |
|------|-----------|---------|------|
| L1~L3 | 과잉 일반화/왜곡형 | 1 | 지문의 개념을 사용하되 인과관계를 왜곡 (원인↔결과 뒤바꿈, 필요→충분 조건 오류, 상관→인과 오류) |
| L4~L5 | 유사 정답형 | 1 | 정답 체인의 특정 단계와 비슷하지만 인과 경로가 다른 진술. 체인에 넣으면 논리가 미묘하게 깨짐 |
| L6~L7 | 혼합 | 2 | 왜곡형 1개 + 유사 정답형 1개. 유사 정답형은 체인의 서로 다른 위치에 끼어들 수 있어야 함 |

**금지 패턴:**
- "반드시", "완전히", "모든", "~만이", "전혀" 등 극단적 한정어 사용 금지
- 지문과 정반대 내용으로만 구성된 오답 금지
- 상식만으로 제거 가능한 오답 금지

### 2. 카드 배치 — 이중 셔플

- **id 랜덤 배치**: 정답 체인이 알파벳 순서와 일치하지 않도록. 예: `correct_chain: ['c', 'a', 'e', 'b']`
- **텍스트 재구성**: 카드 텍스트는 지문 문장의 직접 인용이 아닌, 동일 논리를 다른 표현으로 재구성

### 3. 소재

- 교과서적 소재 20% + 수능 기출 스타일 신선한 소재 80%
- 학생이 처음 접하는 소재를 논리만으로 풀 수 있어야 함

### 4. 문제 수량 — 고레벨 강화

| 레벨 | 문제 수 |
|------|---------|
| L1 | 8 |
| L2 | 8 |
| L3 | 8 |
| L4 | 10 |
| L5 | 10 |
| L6 | 12 |
| L7 | 12 |
| **합계** | **68** |

### 5. 지문 길이

| 레벨 | 문장 수 | 비고 |
|------|---------|------|
| L1~L2 | 6~8문장 | 짧고 명확 |
| L3~L4 | 최소 10문장 | 배경 정보와 예시 포함 |
| L5 | 최소 10~12문장 | 대조/역전 논리 포함 |
| L6 | 12~15문장 | 다층 논증 |
| L7 | 13~15문장 | 복잡한 논리 + 전문 개념 |

### 6. 주제

- 현행 5개 (humanities, social, science, tech, arts) 유지
- 추후 주제 추가 예정
- 레벨별 균등 배분 강제 안 함

## 생성기 프롬프트 개선 (`lib/papers/generator.ts`)

### 변경 사항

1. **오답 지시 강화**: 레벨별 오답 유형을 프롬프트에 명시
2. **카드 id 셔플**: "정답 카드의 id 순서가 정답 체인 순서와 일치하지 않도록 배치"
3. **텍스트 재구성**: "카드 텍스트는 지문의 문장을 그대로 복사하지 말고, 동일한 논리를 다른 표현으로 재구성"
4. **지문 길이 조정**: L3 이상 최소 10문장, L6~7은 12~15문장
5. **논문 기반 명시**: 원문(논문)에서 핵심 논증 구조를 추출하되, 지문은 수능 비문학 스타일로 재구성

```typescript
const LEVEL_GUIDE = {

  2: { slots: 3, distractors: 1, distractorType: 'distortion', hints: 3, passageLen: '200~300자 (6~8문장)', style: '부분 추상 개념' },
  3: { slots: 4, distractors: 1, distractorType: 'distortion', hints: 3, passageLen: '300~450자 (최소 10문장)', style: '수식어/종속절' },
  4: { slots: 4, distractors: 1, distractorType: 'pseudo-answer', hints: 3, passageLen: '350~500자 (최소 10문장)', style: '모의고사 수준' },
  5: { slots: 5, distractors: 1, distractorType: 'pseudo-answer', hints: 3, passageLen: '400~550자 (10~12문장)', style: '대조/역전 논리' },
```

### 오답 유형별 프롬프트 지시

```
distortion (왜곡형):
- 지문에 등장하는 개념과 용어를 사용하되, 인과관계를 왜곡하세요.
- 원인과 결과를 뒤바꾸거나, 필요조건을 충분조건으로 바꾸거나, 상관관계를 인과관계로 오인시키세요.
- "반드시", "완전히", "모든", "~만이" 등 극단적 표현 사용 금지.
- 지문을 읽지 않고는 오답 여부를 판단할 수 없어야 합니다.

pseudo-answer (유사 정답형):
- 정답 체인의 특정 단계와 비슷하지만 인과 경로가 미묘하게 다른 진술을 만드세요.
- 이 카드를 정답 체인에 넣으면 논리가 깨지지만, 얼핏 보면 들어맞는 것처럼 보여야 합니다.
- 정답 카드와 텍스트 길이, 문체가 동일해야 합니다.

mixed (혼합):
- 왜곡형 1개 + 유사 정답형 1개를 만드세요.
- 유사 정답형은 체인의 서로 다른 위치에 끼어들 수 있는 것으로 만드세요.
```
